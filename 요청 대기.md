toddler-checklist.tsx와 quiz.tsx 두 컴포넌트가 내부적으로 따로 놀고 있는 데이터 구조와 채점 로직을 하나로 통합하려고 해. 기존의 예쁜 비주얼(UI)은 그대로 유지하되, 내부 로직은 완전히 아래의 통합된 JSON 규격을 따르도록 수정해 줘.

[통합 데이터 타입 (workpad-data.ts 등에 정의)]
두 컴포넌트 모두 이 AssessmentQuestion 배열을 props로 받아야 해.

TypeScript
export type AssessmentQuestion = {
  question_id: string; // 질문 고유 ID
  skill?: string;      // 문제 유형 (추론, 어휘 등)
  passage?: string;    // 예문 (없을 수 있음)
  prompt: string;      // 질문 내용
  options: { 
    label: string;     // 보기 텍스트
    value: string;     // 보기 고유 값 (a, b, c 등)
    image_url?: string | null; // 그림 퀴즈용 이미지 (UI 반영 필요)
  }[];
  level?: number;      // 난이도 (프론트에선 렌더링 안 함)
  answer?: string;     // 정답 (프론트에선 렌더링 안 함)
}
[수정 사항 1: 공통 onComplete 콜백 통일]

두 컴포넌트 모두 자체적으로 정답 수(correct)를 계산하거나 점수(score)를 매기는 가짜 채점 로직을 완전히 삭제해. 백엔드에서 채점할 거야.

대신, 유저가 선택한 답안 객체 전체를 그대로 부모에게 넘겨줘.

즉, onComplete: (answers: Record<string, string>) => void 형태로 통일해. (예: { q_001: "a", q_002: "c" })

[수정 사항 2: toddler-checklist.tsx 업데이트]

기존의 하드코딩된 더미 대신 questions: AssessmentQuestion[]를 props로 받아 세로형 리스트로 렌더링해 줘.

옵션을 고를 때 score를 찾는 로직을 전부 지우고, 단순히 선택된 value만 상태에 저장해.

[수정 사항 3: quiz.tsx 업데이트]

한 문제씩 넘기는 슬라이더 형태의 기존 비주얼을 유지해.

중요 (그림 퀴즈 UI 추가): options 배열을 렌더링할 때, 객체 안에 image_url이 존재한다면 label 텍스트 위나 옆에 해당 이미지를 예쁜 정사각형 썸네일로 렌더링해 줘. image_url이 null이거나 없으면 기존처럼 텍스트만 보여주면 돼.

마지막 문제에서 '결과 확인하기'를 누르면, 내부에서 계산하던 bySkill 로직을 지우고 저장된 answers 객체만 부모로 올려보내.
