## 구글 계정 로그인 (front -> back)
### DB에 해당 유저가 없으면 새로 생성하고 있으면 통과한다.

- 엔드포인트: POST /api/v1/users/sync
- Request :
{
  "user_id": "supabase-uuid-1234",
  "email": "user@gmail.com",
  "name": "홍길동"
}

- Response: { "status": "ok", "message": "User synced" }

## 폼 입력 -> 동화/퀴즈 생성 -> 채점 -> 결과 반영 파트 (back -> front)

### 동화(+퀴즈) 요청 (front -> back)

#### 분기 1: 폼 입력 후 맞춤형 동화 및 퀴즈 생성 요청
- 목적: 아이의 폼 입력을 바탕으로 새로운 동화와 퀴즈를 생성
- 엔드포인트: POST /api/v1/stories/generate
- Request:
{
  "profile_id": "prof_abc123",
  "assessment_type": "posttest", 
  "protagonist_name": "도란이",    
  "favorite": "공룡",            
  "today_event": "오늘 유치원에서 놀았어요." 
}

#### 분기 2: 배치고사 문제 생성 요청 (동화 제외)
- 목적: 동화 없이 아이의 수준을 파악하기 위한 배치고사 문제 출제
- 엔드포인트: POST /api/v1/assessments/generate
- Request:
{
  "profile_id": "prof_abc123",
  "assessment_type": "pretest" 
}

#### 분기 3: 보관함(라이브러리)에 저장된 특정 동화 요청 (퀴즈 제외)
- 목적: 이전에 생성하여 DB에 저장해둔 동화를 다시 읽기 위함
- 엔드포인트: GET /api/v1/stories/saved/{story_id}?profile_id={profile_id}
- Request: (URL Path와 Query Parameter로 전달하므로 Request Body 없음)

#### 분기 4: 자체 제공(기본) 전래동화 요청 (퀴즈 제외, 비로그인 가능)
- 목적: 로그인 없이 또는 기본 라이브러리에서 제공하는 기성 동화 열람
- 엔드포인트: GET /api/v1/stories/default/{story_id}?profile_id=guest
- Request: (URL Path와 Query Parameter로 전달하므로 Request Body 없음)


### 서버에서 동화 + 퀴즈 전송(공통 규격) - (back -> front)

- 배치고사 시에는 퀴즈만 묶어서. 동화 생성시엔 동화와 퀴즈를 묶어서. 라이브러리 조회시나 기본 제공 동화의 경우엔 동화만. 나머지는 null이나 빈 배열 처리

- Response: 
{
  // 평가 유형: "pretest"(아동 배치고사/재시험), "posttest"(동화 퀴즈), "checklist"(영유아 부모용)
  "assessment_type": "pretest", 
  
  // ▼ 동화 파트 (posttest일 경우에만 값이 들어가고, 배치고사나 그냥 보는 테스트면 null/빈 배열 처리)
  "story_id": "story_102938",
  "title": "공룡 티라노의 사과",
  "theme": "용기",
  "pages": [
    {
      "page_number": 1,

      "text": "옛날 옛적, 용감한 공룡 티라노가 살았어요."
    }
  ],
  
  // ▼ 퀴즈 파트 (배치고사, ) - 프론트엔드의 AssessmentQuestion 타입을 정확히 따름
  "quizzes": [
    {
      "question_id": "q_045",
      "skill": "이해",                   // 문제 유형 (어휘, 이해, 추론 등)
      "passage": "토끼가 깡충깡충 뛰어 풀밭으로 갔어요.", // 예문 (체크리스트 등 예문이 없으면 null)
      "prompt": "토끼가 어디로 갔는지 고르세요.",         // 실제 질문 내용
      "options": [
        { 
          "label": "강가",

        },

        }

      "level": 12,                      // 난이도 가중치 (추후 서버 채점 시 프론트에 안 줘도 됨)

### 문제나 체크리스트 완료 후, 프론트에서 서버로 보낼 결과 내용 

- 엔드포인트 : POST /api/v1/assessments/submit
- Request: 
{
  "profile_id": "prof_123abc",          // 퀴즈를 푼 아이의 프로필 ID
  "assessment_type": "pretest",         // 출제 시 받았던 평가 유형 그대로 반환
  "story_id": null,                     // 동화 퀴즈였다면 해당 story_id 반환
  
  "results": {
    "total_questions": 5,               // 전체 문제 수
    "correct_answers": 4,               // 맞춘 문제 수 (프론트 1차 채점 결과)
    "details": [
      { 
        "question_id": "q_045", 
        "level": 12,                    // 해당 문제의 난이도
        "selected_value": "b",          // 사용자가 최종 선택한 보기의 value (서버 교차 검증용)
        "is_correct": true              // 프론트엔드 자체 채점 정오답 여부
      },
      { 
        "question_id": "q_046", 
        "level": 14, 
        "selected_value": "a", 
        "is_correct": false 
      }
    ]
  }
}

### 서버에서 채점결과 반영 후 클라이언트로 보낼 진척도 정보

- Response: 
{
  // 측정 종류: "initial"(최초 측정), "post-story"(동화 읽은 뒤), "retest"(재시험)
  "kind": "initial",
  
  // 성적 데이터
  "correct": 4,                         // 맞춘 문항 수
  "total": 5,                           // 전체 문항 수
  "delta": null,                        // 직전 측정 대비 변화량(%). 최초 측정이면 null
  
  // 환산된 최종 레벨 데이터 (프론트 진행 막대 렌더링용)
  "level": 14,                          // 백엔드가 추세/가중치를 반영해 계산한 최종 20단계 중 레벨
  "lowerLabel": "나무 2단계",              // 막대 왼쪽 라벨 (현재 확실히 도달한 단계)
  "upperLabel": "나무 3단계",              // 막대 오른쪽 라벨 (다음 목표 단계)
  "achievement": 65.5                   // 두 단계 사이에서의 상세 달성도 (0~100 사이, 소수점 1자리)
}
