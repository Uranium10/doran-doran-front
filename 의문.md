방금 통화를 해보고 오니, 다음과 같은 **문제점/추가할 점**들이 보이는데, 순서대로 v0에 입력할 프롬프트나 데이터 규격, 엔드포인트 코딩 등을 체계적으로 도와주며 의견 수립을 도와줘야 할 것 같네..나도 지금 올린 문서들을 전부 읽은 건 아니라, 보완해야 할 부분도 말해주면 좋아.

1. 로그인을 하면 바로 자녀 프로필(이하 프로필)을 필수적으로 고르도록 해야 할 듯. 프로필 선택창은 디즈니+ 같이,  이름과 함께 라운드형 프로필들을 보여주고, 맨 끝에 + 모양으로 '새 프로필 생성' 기능을 달아야 할 듯. 프로필에 마우스를 올리면 프로필 구석에 조그맣게 - 표시가 나오고, 정말 삭제할까요? 를 띄우며 제거하는 기능도 구현해야 할 것 같음. 실제 삭제 시에 수행할 서버와의 JSON 통신과  엔드포인트가 필요.

2. 프로필 생성시엔 프로필 이름과 생년월일, 성별, 썸네일 등의 정보만 받아도 될 듯. 

3. 프로필을 고를 경우, 아래 항목들의 경우를 고려하여 해당 프로필의 정보(이름, 나이,  프로필id, 문해력 측정 여부, 문해력 단계)를 세션에 보유해야 할 것 같음. 이를 받아오기 위한 새 규격과 엔드포인트를 추가해야 함.

4. 프로필 생성 후, 동화 생성 서비스를 위해선 문해력 측정이 반드시 선행되어야 함. 프로필 생성을 완료하면 '동화 제공을 위해 문해력 측정이 필요해요. 지금 하러 갈까요?' 등의 메세지를 띄우고, '지금 할게요' 를 누르면 문해력 측정 페이지로 넘어가고, '나중에 할게요'를 누르면 메인 화면으로 돌아가야 함. 

5.  문해력 측정 페이지는 폼 구분이 필요할 것 같음. 예를 들어 영유아기 아동들은 문해력 측정 자체가 불가능함. 따라서 프로필 정보를 불러온 뒤,
    - 일정 연령 이하 영유아의 경우 '**부모님**이 작성해야 해요' 라는 메세지를 먼저 보여줘야 하고, 그 이상의 경우 **아이가 문제를 읽고 풀 수 있도록 해 주세요** 같은 메세지를 보여줘야 할 것 같아.
    - 영유아는 제시된 문서처럼 셀렉트 형태의 폼으로 구성되고, 그 이상 연령은 퀴즈들로 구성되는 분기를 두어야 할 것 같아. 
    - 아마 직접 퀴즈를 풀 나이 중 저연령대는 그림 퀴즈로, 그 이상은 문해 퀴즈 위주로 오겠지?
    - 부모님이 작성하는 폼은 가능하면 한 화면당 여러 항목이 나오게, 아이가 풀어야 하는 퀴즈는 기존의 동화 정독 후 나오는 퀴즈와 동일한 규격으로 맞춰서 가독을 쉽게 만들고, 현재 얼마나 풀었고, 얼마나 남았는지도 표시해야 할 것 같아. 필수 항목의 입력이 덜 되었을 경우 돌아가서 작성할 수 있게도 해야 할 것 같고.
    - 각 분기에 맞게 JSON을 만들어야 할지, 그냥 기존의 quiz JSON을 활용할 지 정해야 할 것 같구나.

6. 로그인한 계정과 별도로, 선택한 프로필의 정보(이름, 문해력 단계 아이콘, 썸네일)도 표시해야 할 듯. 상단 바 혹은 메인 화면에 표시해야 할 것 같아.
- 상단 바에 표시하는 경우:   
    a. 해당 프로필 정보 칸을 누르면 '다른 프로필로 이동, 새싹 2단계' 등의 팝업 메뉴가 보여야 할 것 같음.
    b. '새싹 2단계' 항목은 현재 등록된 문해력 단계를 의미(아직 문서를 다 안읽어서 단계 이름이나 세부 단계가 있을 지는 모르겠는데 세부 단계가 없다면 네가 만들어주거나 해야 할듯.). 아직 문해력 측정 데이터가 없다면, 아직 문해력 데이터가 없어요(메세지), 지금 측정 하기(버튼) 등의 인터페이스가 나와야 하는데, 팝업 메뉴가 그 정도의 정보량을 담을 크기가 될 지, 모바일 화면에서 그런 형태가 가능할 지는 좀 의문.
    c. 좁은 상단 바 내부에 접속된 구글 계정과 프로필을 동시에 표시하는게 과연 맞을까? 프로필을 고르면 프로필만 표시하게끔 하면 되나.. 싶긴 하지만, 구글 로그아웃을 하고 싶다면? 프로필 목록으로 다시 돌아온 뒤에 로그아웃 하게 둘 수는 있다.
- 메인 화면 상단에 표시하는 경우:
     a. 상단 바에 표시하는 것과 대동소이한 구성. 공간 활용은 좋지만, 어디서나 접근하기 난해할 수 있고, 후술하겠지만 현재 '지금 시작하기' 버튼이 프로필 선택 뒤엔 '동화 생성하기' 로 바뀔 수가 있어 서비스의 실질적인 기능인 해당 버튼과 순서 위계에 따라 주목도 경쟁을 해야 함. 좋지는 않은 방법인듯.

7. 프로필이 선택된 상태면 '지금 시작하기' 버튼들이 '동화 만들기' 형태로 대체되어야 함. 단, 해당 버튼을 눌렀을 때 현재 세션에 문해력 데이터가 없는 경우라면, 프로필 생성시에 나타났던 '문해력 측정이 필요해요. 지금 측정할까요?' UI를 보여주며 문해력 측정 페이지로 이동해야 함.

8. 회원이 '라이브러리' 를 눌렀을 때, 생성했던 동화 내용을 불러올 페이지가 필요함. 격자형/리스트형 토글으로 보여주고, 썸네일, 제목, 생성 날짜, 당시 문해력 단계 등을 표시해야 할 듯.
