[목표1 - 로그인 및 프로필 통신 로직 교환]
현재 UI에 구현된 더미 데이터 로직(useProfile 컨텍스트 내부의 상태 관리)을 모두 실제 fetch API 호출로 교체해 줘.

[API 기본 설정]
프론트엔드 전역에서 사용할 API Base URL은 NEXT_PUBLIC_API_BASE_URL 환경 변수에서 가져오고, 값이 없으면 "http://localhost:8000/api/v1"을 폴백으로 사용해 줘.

[구현해야 할 통신 흐름]
1. 유저 동기화 (auth-redirect-gate.tsx, auth-button.tsx 등): 
Supabase 로그인에 성공해서 session을 가져왔을 때, POST /users/sync 를 호출해 백엔드 DB와 동기화해.
- payload: { "user_id": session.user.id, "email": session.user.email, "name": session.user.user_metadata.name ?? session.user.user_metadata.full_name }

2. 프로필 목록 조회 (profile-context.tsx, profile-picker.tsx):
프로필 선택 화면 진입 시, GET /profiles?user_id={session.user.id} 를 호출하고 응답받은 { "profiles": [...] } 배열로 상태를 렌더링해 줘.

3. 프로필 생성 (profile-picker.tsx, profile-form-modal.tsx):
새 프로필 폼 제출 시 POST /profiles 를 호출해.
- payload: { "user_id": session.user.id, "name": "...", "birth_date": "...", "avatar_url": "..." }
- 성공 후 응답받은 프로필 객체를 목록 상태에 추가.

4. 프로필 수정 (app-header.tsx, profile-form-modal.tsx):
수정 폼 제출 시 PATCH /profiles/{profile_id} 를 호출해.
- payload: { "name": "...", "birth_date": "...", "avatar_url": "..." }
- 성공 후 응답받은 객체로 currentProfile 및 목록 상태를 업데이트.

5. 프로필 삭제 (profile-picker.tsx, confirm-modal.tsx):
삭제 모달에서 확인 시 DELETE /profiles/{profile_id} 를 호출해. 
- 성공하면 목록 상태에서 해당 프로필 제거.

동화 이미지가 당장 생성되진 않을 예정이라, 동화 데이터가 왔을 때, 해당 페이지에 이미지 url이 탑재되지 않거나, 읽을 수 없는 경우 기존에 쓰던 더미 데이터를 사용해야 할 것 같아.
