우리가 만들고 있는 도란도란 서비스의 프론트엔드 비즈니스 로직과 API 연동을 전면 업데이트하려고 해. 아래 3가지 목표를 완벽하게 반영해서 관련된 파일들을 수정해 줘.

[목표 0] 유아/아동 분기 기준 변경 (나이 -> 레벨)
1. `lib/levels.ts`에서 영유아(toddler)와 아동(child)을 구분하는 기준을 나이(72개월)에서 '프로필 레벨'로 변경해. 
   - `literacyMode` 함수가 `birthDate` 대신 `level: number | null`을 파라미터로 받도록 수정해.
   - `level`이 `null`이거나 `8` 미만이면 `"toddler"`, `8` 이상이면 `"child"`를 반환하게 해.
2. `app/literacy/page.tsx` (또는 해당 로직이 있는 파일)에서 `literacyMode(currentProfile.birth_date)`로 호출하던 부분을 `literacyMode(currentProfile.level)`로 수정해서 에러가 없도록 맞춰 줘.

[목표 1] 동화 열람 완료 후 분기 처리 (Dashboard & PopupBook)
`page.tsx` (또는 대시보드 컴포넌트)의 `handleBookFinish` 함수와 상태(State) 로직을 아래 3가지 케이스에 맞게 전면 수정해 줘. `assessment.assessment_type`과 `assessment.quizzes` 배열을 기준으로 분기해야 해.

1. 맞춤 동화 생성 (posttest) & 퀴즈가 없는 경우 (`quizzes.length === 0`):
   - 퀴즈 화면(`post-quiz`)으로 넘어가지 마.
   - 화면을 대시보드(`home`)로 전환하면서, 부모님을 위한 알림 모달을 띄워 줘.
   - 모달 내용: "동화를 다 읽었어요! 난이도 변경을 원하시면 문해력 재측정을 진행해 주세요." (확인 버튼 클릭 시 모달 닫힘)
2. 맞춤 동화 생성 (posttest) & 퀴즈가 있는 경우 (`quizzes.length > 0`):
   - 기존처럼 퀴즈 화면(`post-quiz`)으로 이동해.
3. 라이브러리/기본 동화 열람 (assessment_type이 "library" 또는 "readonly"인 경우):
   - 퀴즈 화면으로 넘어가지 마.
   - 컴포넌트 뷰를 `home`으로 돌리거나 `router.back()`을 호출해서 원래 있던 곳(이야기 책장 등)으로 자연스럽게 돌아가게 해.

[목표 2] 실제 백엔드 API 연동 (fetch 적용)
`lib/workpad-data.ts`와 `library-rows.tsx`의 더미 로직을 실제 API `fetch`로 교체해. 내가 정의한 통신 규격을 철저히 지켜야 해.

1. `workpad-data.ts` 수정:
   - `generateAssessment` 함수: `(profileId: string, assessmentType: string, input: StoryInput)`을 파라미터로 받고, `POST /api/v1/stories/generate`로 요청하도록 fetch 구현.
   - `generatePretest` 함수 (신규 추가): `(profileId: string)`을 받고 `POST /api/v1/assessments/generate`로 요청하여 `AssessmentPayload` 반환. (`assessment_type: "pretest"` 포함)
   - `submitAssessment` 함수: `POST /api/v1/assessments/submit`로 요청하도록 fetch 구현 (요청 Body에 `submission` 객체 전송).
2. `library-rows.tsx` 수정:
   - 더 이상 `@/lib/folktales`의 하드코딩된 정적 데이터를 쓰지 마.
   - 컴포넌트가 마운트될 때 `GET /api/v1/library/folktales`를 호출해서 받아온 `folktales`와 `rows` 데이터로 화면을 렌더링해 줘. (로딩 중일 때는 예쁜 스켈레톤 UI 렌더링 포함)
