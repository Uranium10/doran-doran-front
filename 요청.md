[목표 1: 로그인 및 프로필 통신 로직 교환]
현재 UI에 구현된 더미 데이터 로직(useProfile 컨텍스트 내부의 상태 관리)을 모두 실제 비동기(fetch) API 호출로 교체해 줘.

[API 기본 설정]
프론트엔드 전역에서 사용할 API Base URL은 `NEXT_PUBLIC_API_BASE_URL` 환경 변수에서 가져오고, 값이 없으면 폴백으로 `"https://doran-doran.onrender.com/api/v1"`을 사용해 줘.

[구현해야 할 통신 흐름 (전체 로딩/에러 처리 포함)]
* 모든 API 호출 시, 중복 클릭을 막기 위해 버튼을 로딩 상태(disabled)로 만들고 에러 발생 시 콘솔에 출력해 줘.
* `profile-context.tsx`의 상태 업데이트 함수들이 비동기(Promise)를 지원하도록 수정해 줘.

1. 유저 동기화 (auth-redirect-gate.tsx, auth-button.tsx 수정): 
Supabase 로그인에 성공해서 session을 가져왔을 때, `POST /users/sync` 를 호출해 백엔드 DB와 동기화해.
- payload: { "user_id": session.user.id, "email": session.user.email, "name": session.user.user_metadata.name ?? session.user.user_metadata.full_name }

2. 프로필 목록 조회 (profile-context.tsx, profile-picker.tsx 수정):
프로필 선택 화면 진입 시, `GET /profiles?user_id={session.user.id}` 를 호출하고 응답받은 `{ "profiles": [...] }` 배열로 `profiles` 전역 상태를 덮어씌워 렌더링해 줘.

3. 프로필 생성 (profile-form-modal.tsx, profile-picker.tsx 수정):
새 프로필 폼 제출 시 `POST /profiles` 를 호출해.
- payload: { "user_id": session.user.id, "name": "...", "birth_date": "...", "avatar_url": "..." }
- 성공 후 응답받은 프로필 객체를 목록 상태에 추가.

4. 프로필 수정 (app-header.tsx, profile-form-modal.tsx 수정):
수정 폼 제출 시 `PATCH /profiles/{profile_id}` 를 호출해.
- payload: { "name": "...", "birth_date": "...", "avatar_url": "..." }
- 성공 후 응답받은 객체로 currentProfile 및 목록 상태를 업데이트.

5. 프로필 삭제 (confirm-modal.tsx, profile-picker.tsx 수정):
삭제 모달에서 확인 시 `DELETE /profiles/{profile_id}` 를 호출해. 
- 성공하면 목록 상태에서 해당 프로필 제거.
