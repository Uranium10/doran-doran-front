우리가 만들고 있는 도란도란 서비스의 프론트엔드 비즈니스 로직과 API 연동을 전면 업데이트하려고 해. 통신규격.md 파일과 아래 4가지 목표들을 완벽하게 반영해서 관련된 파일들을 수정해 줘.

[목표 0] 유아/아동 분기 기준 변경 (나이 -> 레벨)
※ 전제: 문해력 레벨은 백엔드가 계산하는 1~20단계 체계이며, 소수점(예: 17.5)으로 내려올 수 있다.
   문제은행은 1~8단계=부모 체크리스트, 9~20단계=아동 4지선다로 구성되어 있다.

1. `lib/levels.ts`에서 영유아(toddler)와 아동(child)을 구분하는 기준을 나이(72개월)에서 '프로필 레벨'로 변경해.
   - `literacyMode` 함수가 `birthDate` 대신 `level: number | null`을 파라미터로 받도록 수정해.
   - `level`이 `null`이거나 `9` 미만이면 `"toddler"`, `9` 이상이면 `"child"`를 반환하게 해.
     (문제은행상 4지선다가 9단계부터 시작하므로 경계는 9다.)
2. `getStageInfo` 등 레벨→단계 환산 로직을 12단계가 아닌 20단계 체계로 맞춰라.
   단계 라벨은 씨앗 6칸 / 새싹 5칸 / 나무 5칸 / 숲 4칸이며, 소수 레벨은 내림(floor) 처리한다.
   (측정 결과 화면의 단계 표시는 서버가 준 lowerLabel/upperLabel을 그대로 신뢰한다.)

[목표 1] 동화 열람 완료 후 분기 처리 (Dashboard & PopupBook)
`page.tsx` (또는 대시보드 컴포넌트)의 `handleBookFinish` 함수와 상태(State) 로직을 아래 3가지 케이스에 맞게 전면 수정해 줘. `assessment.assessment_type`과 `assessment.quizzes` 배열을 기준으로 분기해야 해.

1. 맞춤 동화 생성 (posttest) & 퀴즈가 없는 경우 (`quizzes.length === 0`):
   - 퀴즈 화면(`post-quiz`)으로 넘어가지 마.
   - 화면을 대시보드(`home`)로 전환하면서, 부모님을 위한 알림 모달을 띄워 줘.
   - 모달 내용: "동화를 다 읽었어요! 난이도 변경을 원하시면 문해력 재측정을 진행해 주세요." (확인 버튼 클릭 시 모달 닫힘)
2. 맞춤 동화 생성 (posttest) & 퀴즈가 있는 경우 (`quizzes.length > 0`):
   - 기존처럼 퀴즈 화면(`post-quiz`)으로 이동해.
3. 라이브러리/기본 동화 열람 (assessment_type이 "library" 또는 "readonly"인 경우):
   - 퀴즈 화면으로 넘어가지 마.
   - 컴포넌트 뷰를 `home`으로 돌리거나 `router.back()`을 호출해서 원래 있던 곳(이야기 책장 등)으로 자연스럽게 돌아가게 해.

[목표 2] 실제 백엔드 API 연동 (fetch 함수 구현)
`lib/workpad-data.ts`와 `library-rows.tsx`의 더미 로직을 실제 API `fetch`로 교체해.

1. `workpad-data.ts` 수정:
   - `generateAssessment` 함수: `(profileId: string, assessmentType: string, input: StoryInput)`을 파라미터로 받고, `POST /api/v1/stories/generate`로 요청하도록 fetch 구현.
   - `generatePretest` 함수 (신규 추가): `(profileId: string)`을 받고 `POST /api/v1/assessments/generate`로 요청하여 `AssessmentPayload` 반환. (`assessment_type: "pretest"` 포함)
   - `submitAssessment` 함수: `POST /api/v1/assessments/submit`로 요청하도록 fetch 구현 (요청 Body에 `submission` 객체 전송).
2. `library-rows.tsx` 수정:
   - 더 이상 `@/lib/folktales`의 하드코딩된 정적 데이터를 쓰지 마.
   - 컴포넌트가 마운트될 때 `GET /api/v1/library/folktales`를 호출해서 받아온 `folktales`와 `rows` 데이터로 화면을 렌더링해 줘. (로딩 중일 때는 예쁜 스켈레톤 UI 렌더링 포함)

[목표 3] 측정 페이지 및 보관함(이야기 책장) 연동
1. `app/literacy/page.tsx` 연동:
   - 파일 상단의 정적 변수 `const pretestAssessment = buildPretestAssessment()`를 지워 줘.
   - 아동 모드(child)일 경우, 컴포넌트 마운트 시 새로 만든 `generatePretest(currentProfile.id)`를 비동기로 호출해서 배치고사 문제를 받아와. 
   - 문제를 받아오는 동안에는 "문제를 준비 중이에요..."라는 로딩 UI(스피너 등)를 보여줘.
2. 이야기 책장(보관함) 연동:
   - `/library` 화면 컴포넌트(만약 없다면 새로 생성)에서 마운트 시 `GET /api/v1/stories?profile_id={currentProfile.id}`를 호출해.
   - 받아온 `stories` 배열을 활용해 기존에 읽은 동화 목록을 카드 형태로 보여줘.
   - 카드 클릭 시 해당 동화 데이터를 들고 `PopupBook` 컴포넌트로 넘어가서 읽을 수 있게 해줘 (이때 assessment_type은 "library"로 취급).

※ 모든 fetch 로직은 try-catch문으로 감싸고, 만약 서버 에러가 나면 앱이 뻗지 않도록 '서버와 연결할 수 없어요'라는 Toast 알림을 띄우고 이전 화면으로 안전하게 돌아가도록 에러 핸들링을 추가해 줘.

[목표 4] 결과 화면 (LiteracyResultView) 실제 데이터 바인딩
이제 백엔드 API 구현이 완료되었으니, 더미 데이터를 걷어내고 실제 통신을 연결해 줘.
1. `POST /api/v1/assessments/submit` API를 호출하고 받은 응답(`level`, `delta`, `lowerLabel`, `upperLabel`, `achievement`)을 상태(State)에 저장해.
2. `components/literacy-result.tsx`에 이 데이터를 Props로 넘겨서 게이지 바, 현재 티어(단계), 성장 수치(+%)가 실제 서버 결과값을 바탕으로 정확하게 렌더링되도록 구현해 줘.

[목표 5] 비로그인(게스트) 처리 및 라이브러리(기성 동화) 연동
1. 게스트 방어 로직: 현재 프로필이 게스트(`profileId`가 "guest"로 시작하거나 미로그인)일 경우, 동화 생성 폼 접근을 막아줘.
2. 기성 동화 라이브러리 목록 렌더링 (`library-rows.tsx`): 
   - 컴포넌트가 마운트될 때 `GET /api/v1/library/folktales`를 호출해서 받아온 `rows`와 `folktales` 데이터를 상태(State)에 저장해.
   - 기존의 정적 데이터 파일(`@/lib/folktales`) 임포트를 제거하고, 받아온 API 데이터를 바탕으로 UI를 렌더링해.
   - ⚠️ [데이터 필드 매핑 주의]: 기존 하드코딩된 변수명과 API 응답 필드가 달라! 
     - API의 `story_id` -> 기존 `id` 역할 (byId 함수 등에 적용)
     - API의 `thumbnail` -> 기존 `cover` (이미지 src)
     - API의 `description` -> 기존 `subtitle`
     - API의 `theme` -> 기존 `ageRange` 자리에 표시
     - API의 `items` (rows 배열 안) -> 기존 `ids` 역할
     위 매핑 규칙을 적용해서 `Card` 컴포넌트와 렌더링 로직의 변수명을 알맞게 수정해 줘.
3. 기성 동화 읽기 모드 (가장 중요):
   - `library-rows.tsx`의 카드 컴포넌트에 하드코딩된 `<Link href="/dashboard">`를 뜯어내고 `onClick` 이벤트로 변경해.
   - 카드를 클릭하면 `GET /api/v1/stories/default/{story_id}` API를 호출해.
   - 데이터를 성공적으로 받아오면 별도의 페이지 이동(라우팅) 없이, 해당 컴포넌트 내에서 상태(State)를 변경하여 기존 목록 대신 `<PopupBook />` 컴포넌트를 화면에 띄워줘.
   - 팝업북을 다 읽고 나면(또는 닫기 버튼을 누르면) 퀴즈 창은 띄우지 말고 다시 라이브러리 목록으로 돌아오게 상태를 초기화해 줘.

[목표 6] 백엔드 엔드포인트 최종 연결 및 통합
1. `통신규격.md`에 정의된 모든 API Endpoint(`POST /api/v1/stories/generate`, `POST /api/v1/assessments/generate` 등)가 프론트엔드의 `api.ts` 또는 서비스 로직에 정확히 연결되었는지 확인해 줘.
2. CORS 에러 방지를 위해 fetch 요청 시 API 주소와 헤더(Content-Type: application/json)가 올바르게 세팅되었는지 점검하고 최종적으로 코드를 통합해 줘.

## 추가 지침
1. submit 응답의 `delta`는 퍼센트가 아니라 **레벨 단위**(예: +0.35)이며, 최초 측정 시 `null`이다.
   결과 화면에서 `+%`로 표시하지 말고 "레벨 변화량"으로 표시하되, `delta`가 null이면 성장 배지를 숨긴다.
2. `assessment_type`은 서버가 직접 내려준다(saved→"library", default→"readonly").
   프론트가 임의로 주입하지 말고 받은 값을 그대로 분기에 쓰고,
   프론트 `AssessmentType` 타입 유니온에 `"library" | "readonly"`를 추가한다.
