[목표1 - 로그인 요청 교환]
로그인 및 프로필 관리 UI에 백엔드 API 통신 로직을 연결하려고 해. 백엔드 서버의 IP와 포트가 확정되었어. 기존의 더미 데이터 로직(예: useProfile 컨텍스트 내부의 로컬 스테이트 추가/삭제 등)을 모두 실제 fetch API 호출로 교체해 줘.

[API 기본 설정]
프론트엔드 전역에서 사용할 API Base URL을 NEXT_PUBLIC_API_BASE_URL 환경 변수에서 가져오면 돼.

[구현해야 할 통신 흐름]

유저 동기화 (auth-redirect-gate.tsx 또는 auth-button.tsx 수정): >    Supabase 로그인에 성공해서 session을 가져왔을 때, POST /users/sync 엔드포인트로 유저 정보를 보내 백엔드 DB와 동기화해 줘.

payload: { "user_id": session.user.id, "email": session.user.email, "name": session.user.user_metadata.name }

프로필 목록 조회 (profile-context.tsx 또는 profile-picker.tsx 수정):
프로필 선택 화면에 진입하면, Supabase의 session.user.id를 이용해 GET /profiles?user_id={id}를 호출하고 그 결과를 화면에 렌더링해 줘.

프로필 생성 (profile-picker.tsx 수정):
새 프로필 폼을 제출할 때 POST /profiles를 호출해 줘.

payload: { "user_id": session.user.id, "name": "...", "birth_date": "...", "avatar_url": "..." }

성공 후 응답받은 프로필 객체를 프로필 목록 상태에 추가해.

프로필 삭제 (profile-picker.tsx 수정):
삭제 모달에서 확인을 누르면 DELETE /profiles/{profile_id}를 호출해. 성공하면 목록 상태에서 제거해 줘.
